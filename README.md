# GrewMeet 스터디 서비스 요구사항
## 대시보드
1. 상단 날짜는 현재 날짜 기준으로 표시 되어야 한다. 
2. 사용자는 연속 출석, 스터디 일정, 참여중인 방을 한 눈에 볼 수 있어야 한다.
3. 일정이 있을 경우 가장 가까운 일정이 대시보드에 표시(스터디명,시간) 되어야 한다. \
   ```일정 모아보기를 눌러 모든 일정을 한 눈에 볼 수 있어야 한다.```
4. 화분 이미지는 출석일 기준으로 ```0~30일, 31~60일, 61~90일, 90~``` 순으로 성장형 이미지로 변경되어야 한다.

## 방목록
1. 사용자는 방 제목과 방장 이름으로 스터디방 검색이 가능해야 한다.
2. 사용자는 스터디명, 카테고리, 모집 인원, 스터디 규칙, 공지사항, 비밀번호를 입력하여 스터디방을 생성할 수 있어야 한다.\
   ``스터디 인원은 최대 8명이며, 비밀번호는 선택 사항이다.``\
   ``또한 비밀번호가 걸려있는 방은 자물쇠 아이콘으로 표시가 되어야한다.``\
   ``방 목록에는 스터디명, 카테고리명, 방 규칙, 생성일, 참여 인원 수가 표시 되어야한다.``
3. 사용자는 방 접속 시 안내 문구와 함께 접속할 지 선택이 가능해야 한다.

## 방목록 -> 채팅방 접속 시
1. 메세지 전송 시 발신자는 오른쪽 초록 말풍선, 수신자는 왼쪽 흰색 말풍선으로 분리된다.\
   ``말풍선 안에는 수신자 명, 메세지 내용, 보낸 시간이 표시 되어야한다.``
2. 사용자는 일정 생성이 가능해야한다.\
   ``방장만 일정 생성이 가능하다.``\
   ``일정 시간, 제목, 진행 시간, 진행 방식(온라인, 오프라인)을 입력해 공지를 띄운다.``\
   ``1채팅방 1일정으로 제한된다.``
3. 사용자는 일정 투표가 가능해야한다.\
   ``방장 포함 그 외에 인원들은 공지를 눌러 투표가 가능하고, 과반 수 찬성일 시 일정이 등록, 반대일 시 일정이 자동 취소가 된다.``\
   ``개개인마다 1인 1투표이며, 중복 투표는 불가능하다.``\
   ``투표는 하루동안 진행되며, 투표 후에는 누가 찬성했는지 반대했는지 알 수 있다.(인원 체크를 위한 익명X)``
4. 사용자는 이미지 전송이 가능해야한다.

## 더보기
1. 다른 서비스로 이동할 수 있는 버튼이 있어야한다.
2. 설정에서 앱 설정이 가능하도록 해야한다.

# API(BACK ENDPOINT)

## User
1. 모든 유저 조회 -> ``/api/user``(GET)
2. 특정 유저 조회(id) -> ``/api/user/{id}``(GET)
3. 특정 유저 조회(email) -> ``/api/user/{email}``(GET)
4. 회원 생성 -> ``/api/user``(POST)
5. 회원 삭제 -> ``/api/user/{id}``(DELETE)
6. 관리자로 변경 -> ``/api/user/{email}``(PUT)
7. 출석 체크 -> ``/api/user/{id}/attendance``(POST)
8. 방 참여 -> ``/api/user/{id}/join-room``(POST)
9. 방 퇴장 -> ``/api/user/{id}/leave-room``(POST)

## Study
1. 모든 스터디룸 조회 -> ``/api/study``(GET)
2. 특정 스터디룸 조회(name) -> ``/api/study/{studyname}``(GET)
3. 스터디룸 생성 -> ``/api/study``(POST)
4. 스터디룸 삭제 -> ``/api/study/rooms/{studyRoomId}``(DELETE)
5. 스터디룸 참가 -> ``/api/study/rooms/join``(POST)
6. 모든 카테고리 조회 -> ``/api/study/category``(GET)
7. 카테고리 생성 -> ``/api/study/category``(POST)
8. 카테고리 삭제 -> ``/api/study/category/{categoryId}``(DELETE)
9. 스터디룸 규칙 설정 및 수정 -> ``/api/study/rule/{studyRoomId}``(PATCH)
10. 스터디룸 규칙 삭제 -> ``/api/study/rule/{studyRoomId}``(DELETE)
